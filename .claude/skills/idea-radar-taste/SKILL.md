# Idea Radar — разбор вкуса (👍/👎 → preferences.json)

Замыкает петлю вкуса: клики в дашборде → паттерны/причины/гео → `preferences.json`, который читает прогон.
Детали потока и формы элементов — `.claude/rules/taste-loop-rules.md`.

## Вход

Один из:
- JSON-экспорт `idea-radar-taste-export-v1` (пользователь нажал «⬇ Экспорт вкуса» — отдал текстом или файлом
  `idea-radar-taste.json`). **Предпочтительно** — там есть поле `why` (причины).
- Сырой дамп localStorage `idearadar2_fb` (`{ "12":"up", "9":"down" }`) — тогда причин нет, выводи их сам.

Если входа нет — попроси пользователя нажать в дашборде **«⬇ Экспорт вкуса»** и прислать результат. Без реальных
отметок вкус НЕ выдумывать (см. стоп-правила).

## Шаги

**1. Сопоставь.** Для каждой отметки найди карточку в `findings.json` по `id`. Собери по каждой: `vote`, `why`,
и контекст — `niche`, `method`, `kano`, `conf`, оси `s{}`, `title`, `idea`, `pain`, `segment`, `geo-пригодность`.

**2. Выведи паттерны (поперёк категорий, не только ниша/метод).** Ищи, что повторяется в 👍 (аттракторы) и в 👎
(репеллеры) по осям: ниша · метод · Kano · профиль осей (напр. высокий `ease`+`trend`) · тема (B2C-утилита,
делайт-механика, dev/AI-инструмент, контент) · форма JTBD · **гео**. Опирайся на `why` пользователя как на
первичный сигнал причины; где `why` пуст — формулируй гипотезу причины и помечай её как выведенную.

**3. Гео.** Вынеси в `constraints.geo`: что отвергнуто из-за привязки к другой стране (в `reject`/`examples_rejected`).
Политика по умолчанию — РФ ЛИБО локацио-независимое (игры, приложения, международные сервисы, dev/AI). Не плодить
новое поле в `findings.json` — гео остаётся правилом генерации.

**4. Запиши `preferences.json` (v2).** Заполни `attractors`/`repellers` (`{pattern,dimension,weight,evidence_ids}`),
`reasons.likes`/`reasons.dislikes` (короткие строки), производные `liked_niches`/`disliked_niches`/`liked_methods`,
`constraints` (geo + excluded_niches + hard_filters) и `marks` (снимок отметок с `imported`). Обнови `updated` и
`last_export`. **Сливай, не затирай:** новые отметки добавляй к истории `marks`, усиливай/ослабляй `weight`
существующих паттернов, не теряй ранее накопленные причины. Сохрани валидный JSON (распарси перед записью).

**5. Отчёт пользователю.** Кратко: топ-3 аттрактора, топ-3 репеллера, гео-вывод, и 1–2 конкретных смещения для
следующего прогона («буду искать больше X, меньше Y, резать Z по гео»).

## Запись и коммит

- Меняем **только `preferences.json`** (опц. — этот скилл/правила). `findings.json` и `index.html` НЕ трогаем.
- `preferences.json` дашбордом не читается — в `main` его коммитить не обязательно ради Pages; коммить в рабочую
  ветку. Если просят, чтобы прогон сразу подхватил — влей в ветку, с которой он стартует (обычно `main`).

```bash
python3 -m json.tool preferences.json >/dev/null && echo OK   # валидность перед коммитом
git add preferences.json
git commit -m "idea-radar: обновлён вкус (preferences v2) — +N отметок"
```

## НИКОГДА

- **НЕ выдумывать вкус** без реальных отметок. Нет экспорта → работать по гео-политике/excluded по умолчанию.
- **НЕ банить целую категорию из-за одного 👎** — низкий `weight`; жёсткий фильтр только при повторе/явной причине.
- **НЕ выдавать парафраз `why` за цитату** и не приписывать пользователю причину, которой он не высказывал —
  выведенные причины помечать как гипотезы.
