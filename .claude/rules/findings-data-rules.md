# Findings & Dashboard — Стоп-правила

Данные отделены от вида: `index.html` рендерит фиксированный набор полей `findings.json`.
Любая поломка схемы или разметки = пустой/сломанный дашборд на проде.

## НИКОГДА

- **НЕ менять структуру `index.html`** и набор полей, которые он читает (см. контракт ниже). Правки
  вёрстки/стилей — только если явно просят и без изменения контракта данных.
- **НЕ менять схему `findings.json`**: top-level ключи (`schema`, `updated`, `excluded_niches`, `count`,
  `findings`) и форму элемента `findings[]`. Поле `schema` = `idea-radar-findings-v2` не трогать.
- **НЕ считать Diamond Score руками** и НЕ записывать его в данные — его вычисляет `index.html` из осей `s{}`.

## Контракт элемента `findings[]`

Обязательные поля каждого элемента (их читает дашборд):
`id`, `niche`, `method`, `conf`, `kano`, `status`, `title`, `segment`, `jtbd`, `pain`, `existing`, `idea`,
`s{severity, frequency, wtp, reach, gap, trend, ease}` (каждая ось — целое 1–5), `ev[{q, u, t}]` (≥1 элемент).
`demand` — опционально (для метода «спрос»).

Перечисления (ровно эти значения):

- `method`: `спрос` | `боль` | `идея`
- `conf`: `high` | `medium` | `low`
- `kano`: `basic` | `performance` | `delight`
- `status`: `NEW` | `REPEAT` | `GROWING` | `IDEA`
- `ev[].t` — **свободная** короткая метка типа источника (не закрытый список; дашборд рендерит её как текст ссылки).
  Типичные в данных: `стор`, `маркетплейс`, `форум`, `обсуждение`, `отзыв`, `github`, `статья`, `новости`, `обзор`, `отчёт`.

## При добавлении находки

1. `id` = (максимальный существующий id) + 1, уникальный.
2. Дедуп по ключу «нормализованный `title` + `niche`»: повтор не плодить, а обновить `status` (`REPEAT` / `GROWING`).
3. Обновить top-level `count` (= длине `findings`) и `updated` (сегодня, `YYYY-MM-DD`).
4. **Перед коммитом — распарсить JSON** (`python3 -m json.tool findings.json`). Невалидный JSON ломает дашборд молча.

## Зачем

`index.html` обращается к полям напрямую (`d.s.gap`, `d.ev`, `d.niche`, `d.method`, …). Отсутствующее или
переименованное поле = `undefined` в UI или пустая карточка. Diamond Score — производная осей `s{}`;
дублировать его в данных нельзя — он рассинхронизируется с весами дашборда.
