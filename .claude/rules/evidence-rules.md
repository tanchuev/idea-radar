# Evidence — Стоп-правила

Принцип Idea Radar: **доказательство важнее идеи**. Находка без пруфа в `findings.json` не попадает.

## Обязательно

- **Реальный URL + дословная цитата** на каждую находку, в `ev[]`. URL — тот, который реально открыли;
  `q` — дословная фраза/факт со страницы (отзыв 1–2★, тред, заголовок отчёта, число установок).
- **Триангуляция ≥2 независимых источников** на находку (исключение — чистый метод «идея»: статус `IDEA`).
- Для метода «спрос» — подтвердить **объём/тренд** (число запросов, установок, динамика), не только боль.

## НИКОГДА

- **НЕ выдумывать источники, URL и цитаты.** Несуществующий/недоступный пруф → находку удалить,
  а не «дополнить по памяти».
- **НЕ обходить технические ограничения сайтов** (403, пэйволл, антибот). Заблокировано — взять другой источник.
- **НЕ выдавать пересказ за цитату.** В `ev[].q` — дословный текст, не парафраз.

## Red-team перед записью

Каждую находку атаковать: «уже сделано? (назвать конкурентов в `existing`)», «почему до сих пор не сделали —
скрытый барьер (дистрибуция, юнит-экономика, регуляция, платформенный риск)?», «решаемо малой командой,
есть канал первых 100 пользователей?». Непроверяемое и дубли — удалить.

## Зачем

Дашборд показывает цитату и кликабельный URL под каждой карточкой (блок `.ev` в `index.html`). Фейковый
пруф виден сразу и обесценивает весь радар. Цель системы — отсеять «идеи без спроса», поэтому пруф —
входной билет, а не украшение.
