# Idea Radar — ресёрч-прогон

Источник правды промпта — `ROUTINE_PROMPT.md` в корне. Этот скилл — рабочая выжимка. Перед записью
сверься с `.claude/rules/findings-data-rules.md` и `.claude/rules/evidence-rules.md`.

## Шаги

**0. Память.** Прочитай `findings.json` (массив `findings`) и `preferences.json`. Запомни уже найденное
(ключ: нормализованный `title` + `niche`). Под отметки 👍/👎 из `preferences.json` подстрой выбор ниш/методов.

**1. План.** Выбери 3 наименее покрытые ниши из пула; НЕ брать `excluded_niches` (финтех, личные финансы,
учёт бюджета). Чередуй методы между прогонами: demand-first, pain-mining; раз в 2–3 прогона добавляй Ideator.
Пул ниш: 2D-игры; AI/LLM-тулинг; простые продукты (расширения/плагины/микро-SaaS); быт/дом; здоровье/питание;
продуктивность/работа; малый бизнес/самозанятые; онлайн-образование; хобби/контент-креаторы; авто;
родители/дети; питомцы; e-commerce/продавцы; путешествия; недвижимость/ЖКХ.

**2. Скауты (параллельно, по саб-агенту на нишу).** Инструменты: WebSearch, WebFetch.
- **demand-first** (простые продукты): доказать СПРОС (тренды, объём запросов, число установок слабых продуктов
  в Chrome Web Store / сторах) И СЛАБОЕ предложение (низкие рейтинги, реклама, подписка за базовое, заброшено).
  Источники: Google Trends-материалы, Chrome Web Store, App Store / Google Play, Reddit.
- **pain-mining** (RU-ниши): боли по сигнальным фразам и 1–2★ отзывам. Источники: vc.ru, habr, pikabu, dtf,
  drive2, RuStore, a2is, otzovik / irecommend, Яндекс.Маркет.
- **игры**: Steam / Google Play / App Store отзывы, Reddit (r/gaming, r/incremental_games), рыночные отчёты.
- **AI/LLM**: GitHub (issues/trending), Hacker News, Product Hunt, Reddit (r/LocalLLaMA, r/AI_Agents).
- Каждый агент: JTBD «Когда…, я хочу…, чтобы…»; оси 1–5 (severity, frequency, wtp, reach, gap, trend, ease);
  на КАЖДУЮ находку реальный URL (который реально открыл) + дословная цитата/факт. Источники не выдумывать,
  технические ограничения сайтов не обходить — брать другой источник.

**3. Динамика.** Сильный сигнал → запусти ещё скаута глубже по нише; пустую не докапывай. Высокий Diamond Score
+ низкий confidence → агент-валидатор (разбор конкурентов + подтверждение спроса).

**4. Red-team.** Атакуй каждую находку: «уже сделано? (назови конкурентов)», «почему до сих пор не сделали —
скрытый барьер?», «решаемо малой командой и есть ли канал первых 100 пользователей?». Проверь URL/цитаты;
для demand-first подтверди объём/тренд. Дедуп против истории: статус NEW / REPEAT / GROWING. Удали
непроверяемое и дубли.

**5. Синтез.** JTBD; Kano (basic | performance | delight); проставь оси `s{}`. Diamond Score НЕ считать —
дашборд вычисляет сам из осей.

**6. Запись и коммит.** Дописать новые в массив `findings` (`id` = max+1, строго по схеме — см.
`findings-data-rules.md`). Существующим со сменой статуса — обновить `status`. Обновить `count` и `updated`.
Валидный JSON (распарсить перед коммитом). Затем:

```bash
git add findings.json
git commit -m "idea-radar: <YYYY-MM-DD> +N новых находок"
git push origin main
```

Дай в чат дайджест: топ-5 новых/выросших по Diamond Score (title, niche, метод, status, одна строка сути).

## Принципы

Доказательство важнее идеи (реальный URL+цитата ИЛИ доказанный объём спроса); триангуляция ≥2 источников;
сначала боль/спрос (JTBD), потом продукт; `index.html` менять НЕ нужно — дашборд читает `findings.json`.
