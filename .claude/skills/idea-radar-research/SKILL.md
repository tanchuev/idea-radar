# Idea Radar — ресёрч-прогон

Источник правды промпта — `ROUTINE_PROMPT.md` в корне. Этот скилл — рабочая выжимка. Перед записью
сверься с `.claude/rules/findings-data-rules.md` и `.claude/rules/evidence-rules.md`.

> **Экономия контекста:** НЕ читай `findings.json` целиком (~120K токенов — дорого и провоцирует повторы).
> Внешняя память прогона — CLI `.claude/scripts/radar.py`: `summary` (карта), `check` (дедуп), `get <id>`
> (деталь), `add`/`dismiss`/`set-status` (правки). В контекст бери только ВЫВОД команд, не сам файл.
> **Мандат объёма:** каждый прогон дописывает **≥20 новых находок** (после дедупа). Мало — гоняй ещё волны.
> **Массовость:** ищем продукты, полезные МНОГИМ (массовая B2C/SMB-аудитория), НЕ узкий dev/ML/LLM-тулинг.
> **Автономность:** правки данных, git commit/push, создание и **мёрдж** PR разрешены заранее — не спрашивать.

## Служебный CLI (вся работа с большим файлом — через него)

```bash
python3 .claude/scripts/radar.py summary                          # ниши×покрытие, прогоны, dismissed
python3 .claude/scripts/radar.py check "<3–6 слов сути>" --niche "<ниша>"   # дедуп кандидата
python3 .claude/scripts/radar.py get <id> [id ...]                # полная находка по id
python3 .claude/scripts/radar.py add new-findings.json --label "<тема>"     # дописать новые
python3 .claude/scripts/radar.py dismiss --taste idea-radar-taste.json      # 👎 → блоклист + удалить
python3 .claude/scripts/radar.py dismiss --ids 12,34 --why "..."  # точечно отвергнуть
python3 .claude/scripts/radar.py set-status --ids 7 --status GROWING         # дедуп-повтор
python3 .claude/scripts/radar.py validate                         # схема + согласованность индекса
```

## Шаги

**0. Память + вкус (без чтения большого файла).** `radar.py summary` → карта: покрытие ниш (приоритет наименее
покрытым), прогоны, и сколько/каких тем уже **dismissed** (не предлагать). Если в корне есть свежий
`idea-radar-taste.json`: сперва `radar.py dismiss --taste idea-radar-taste.json` (👎«мимо» → dismissed.json +
удаление из findings.json), затем разбери 👍 и причины скиллом `idea-radar-taste` → обнови `preferences.json`
(сливай с историей). Прочитай маленький `preferences.json`: `attractors`/`repellers`/`reasons`/`constraints`
(geo, excluded_niches, hard_filters). Нет вкуса → работай по гео-политике, `excluded_niches` и мандату
массовости (НЕ выдумывай предпочтения). **Гео:** РФ ЛИБО локацио-независимое (игры, приложения/расширения,
международные B2C-сервисы, контент); завязанное на другую страну — переформулировать под РФ/глобально или
отбросить. Детали — `.claude/rules/taste-loop-rules.md`.

**1. План.** Возьми ШИРОКО — 6–8 ниш (приоритет наименее покрытым по `summary`, но можно и покрытые за НОВЫМИ
под-сегментами). НЕ брать `excluded_niches` (финтех, личные финансы, учёт бюджета) и узкий dev/ML/LLM-тулинг.
Сочетай demand-first + pain-mining; почти каждый прогон добавляй Ideator. **Вкус:** к `attractors`, прочь от
`repellers` (не банить нишу из-за одного 👎 — смотри `weight`/причину); на этапе выбора отсекай темы,
жизнеспособные только в другой стране, и dev-инструменты.
Пул ниш: 2D-игры; простые продукты (расширения/плагины/микро-SaaS); быт/дом; здоровье; продуктивность; малый
бизнес/самозанятые; онлайн-образование; хобби/контент-креаторы; авто; родители/дети; питомцы; e-commerce/
продавцы; путешествия; недвижимость/ЖКХ; AI-продукты для ШИРОКОЙ аудитории (потребительские, НЕ dev/ML-тулинг).

**2. Скауты (параллельно, 8+ сразу, по 1–2 на нишу; проси 4–5 находок).** Инструменты: WebSearch, WebFetch.
Источники предпочитай реально открываемые (App Store/Google Play, chrome-stats.com, Steam, reddit, github,
Product Hunt). RU-сайт даёт 403 — бери другой реальный источник, НЕ выдавай сниппет за дословную цитату.
- **demand-first** (простые продукты): доказать СПРОС (тренды, объём запросов, число установок слабых продуктов)
  И СЛАБОЕ предложение (низкие рейтинги, реклама, подписка за базовое, заброшено). Источники: Google Trends,
  Chrome Web Store, App Store/Google Play, Reddit.
- **pain-mining** (RU-ниши): боли по сигнальным фразам и 1–2★ отзывам. Источники: vc.ru, habr, pikabu, dtf,
  drive2, RuStore, a2is, otzovik/irecommend, Яндекс.Маркет.
- **игры**: Steam/Google Play/App Store отзывы, Reddit (r/gaming, r/incremental_games), рыночные отчёты.
- Каждый агент: JTBD «Когда…, я хочу…, чтобы…»; оси 1–5 (severity, frequency, wtp, reach, gap, trend, ease);
  на КАЖДУЮ находку реальный URL (который реально открыл) + дословная цитата/факт. Источники не выдумывать.
- В промпт КАЖДОГО скаута вшей: (1) мандат **массовости** (продукт для широкой аудитории, не узкий dev/ML-тул);
  (2) гео-правило (РФ или локацио-независимое); (3) известные **репеллеры/dismissed** («не предлагай X — это
  „мимо“, причина: …»); (4) аттракторы как ориентир.

**3. Динамика.** Сильный сигнал → ещё скаут глубже; пустую не докапывай. Высокий Diamond Score + низкий
confidence → агент-валидатор (конкуренты + подтверждение спроса).

**4. Red-team + дедуп + вкус/гео.** Для КАЖДОГО кандидата сперва ДЕДУП через скрипт:
`radar.py check "<3–6 слов сути>" --niche "<ниша>"`.
- **⛔ DISMISSED** → DROP (тема отвергнута пользователем).
- **⚠ ВЕРОЯТНЫЙ ДУБЛЬ** → не плодить новую; пометь существующую `radar.py set-status --ids <id> --status
  GROWING` (или REPEAT). Сомнения — `radar.py get <id>` и сравни.
- Иначе — NEW.
Затем атакуй: «уже сделано? (конкуренты в existing)», «скрытый барьер?», «малая команда + канал первых 100?»,
«РФ/локацио-независимо?», «МАССОВЫЙ продукт, а не узкий dev/ML-инструмент?». Проверь URL/цитаты; demand-first —
подтверди объём/тренд.
**DROP только за:** фейк/недоступный пруф; совпадение с dismissed; боль закрыта гос-решением; гео-привязка к
другой стране без переформулировки; узкий dev/ML/LLM-инструмент; нарушение `hard_filters`. **Вкус (не DROP):**
репеллер → переформулируй/занизь; аттрактор → подними. Конкуренты сами по себе — не повод удалять: «тесный
рынок», конкуренты в `existing`, снизь `gap`/`conf`, но ОСТАВЬ. Не выбраковывай агрессивно — цель ≥20.

**5. Синтез.** JTBD; Kano (basic|performance|delight); проставь оси `s{}`. Diamond Score НЕ считать — дашборд сам.

**6. Запись, коммит и мёрдж (автоматически, через скрипт — НЕ редактируй большой файл руками).** Сначала
убедись, что новых находок **≥20**. Запиши их в `new-findings.json` — массив объектов **без** `id`/`run`/`added`
(их проставит скрипт), строго по схеме (см. `findings-data-rules.md`). Затем:

```bash
python3 .claude/scripts/radar.py add new-findings.json --label "<краткая тема прогона>"
python3 .claude/scripts/radar.py set-status --ids <дубли,что выросли> --status GROWING   # если нужно
python3 .claude/scripts/radar.py validate          # должно быть OK
rm new-findings.json
git add findings.json findings-index.json dismissed.json preferences.json   # что менялось
git commit -m "idea-radar: <YYYY-MM-DD> +N новых находок"
git push origin main   # если работаешь на ветке claude/… — push туда, затем СРАЗУ создай PR в main и сам смёржи
```

Дашборд обслуживает `main` — без попадания в `main` (прямой push или мёрдж PR) данные не обновятся. Мёрдж
разрешён заранее, подтверждений не спрашивать.

Дай в чат дайджест: топ-5 новых/выросших по Diamond Score (title, niche, метод, status, одна строка сути).

## Принципы

Доказательство важнее идеи (реальный URL+цитата ИЛИ доказанный объём спроса); триангуляция ≥2 источников;
сначала боль/спрос (JTBD), потом продукт; **массовость** (полезно многим, не узкий dev/ML-тул); работа через
`radar.py` без загрузки всего `findings.json`; `index.html` менять НЕ нужно — дашборд читает `findings.json`.
