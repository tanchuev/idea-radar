# Taste Loop — петля вкуса (👍 «алмаз» / 👎 «мимо»)

Цель: прогон учитывает **что именно** тебе нравится и не нравится — поперёк категорий (не только ниша/метод),
включая **причины** и **гео-пригодность**. Источник вкуса — твои отметки в дашборде.

## Поток данных (как замыкается петля)

```
Дашборд (index.html)                     Репозиторий                         Прогон
  👍/👎 + поле «почему»  ──экспорт──►  idea-radar-taste-export-v1  ──анализ──►  preferences.json (v2)
  (localStorage браузера)             (JSON, вставляешь/кладёшь в репо)        читает ШАГ 0/1/4
```

- Отметки и причины живут в **localStorage браузера** — в репозиторий сами не попадают.
- Кнопка **«⬇ Экспорт вкуса»** в дашборде выгружает их в JSON (`idea-radar-taste-export-v1`): в буфер обмена
  **и** файлом `idea-radar-taste.json`.
- Скилл **`idea-radar-taste`** (или ШАГ 0 прогона) разбирает этот экспорт + карточки `findings.json`,
  выводит паттерны/причины/ограничения и записывает их в **`preferences.json`** (схема `idea-radar-preferences-v2`).
- Прогон читает `preferences.json` и подстраивает поиск.

## Формат экспорта `idea-radar-taste-export-v1`

```json
{"schema":"idea-radar-taste-export-v1","exported":"YYYY-MM-DD","up":N,"down":M,"count":K,
 "marks":[{"id":12,"vote":"up|down","why":"причина пользователя или ''","title":"...","niche":"...",
           "method":"...","kano":"...","conf":"...","run":"...","score":77,"s":{...}}]}
```
`vote`: `up` = «алмаз», `down` = «мимо». `why` — дословная причина, если пользователь её ввёл (может быть пустой).

## Схема `preferences.json` (v2) — формы элементов

- `constraints.geo` — гео-политика (см. ниже). `constraints.excluded_niches` — жёсткие исключения ниш.
  `constraints.hard_filters` — массив строк-правил, нарушение = DROP.
- `attractors` / `repellers` — `[{"pattern":"...","dimension":"niche|method|kano|theme|axis|jtbd|geo|other",
  "weight":1-5,"evidence_ids":[id,...]}]`. `pattern` — человекочитаемое описание паттерна; `weight` — сила
  сигнала; `evidence_ids` — id отметок, на которых паттерн основан.
- `reasons.likes` / `reasons.dislikes` — массив коротких строк: синтез «почему нравится / не нравится».
- `liked_niches` / `disliked_niches` / `liked_methods` — производные списки (удобные срезы для ШАГ 1).
- `marks` — снимок импортированных отметок `[{"id","vote","why","title","niche","run","imported":"YYYY-MM-DD"}]`
  (история, чтобы не переучиваться с нуля и копить причины между экспортами).

## Гео-политика (по умолчанию)

- **Целимся:** рынок **РФ** ЛИБО **локацио-независимое** — игры, полезные приложения/расширения, международные
  сервисы, dev/AI-инструменты, контент.
- **Режем / занижаем:** идеи, завязанные на инфраструктуру/регуляцию/платёжные рельсы **конкретной другой страны**
  (банки только под US, локальные гос-сервисы вне РФ, логистика одной зарубежной страны). Если идея сильная, но
  гео-привязана к другой стране — **переформулировать** под РФ/глобально, иначе отбросить.
- Это объединяет «soft» (занизить `reach`/`conf`, пометить в `existing`) и «hard» (DROP, если без РФ/глобальной
  переформулировки идея нежизнеспособна).

## Как прогон применяет вкус

- **ШАГ 1 (план):** сместить выбор ниш/методов/под-сегментов **к** `attractors` и **прочь от** `repellers`;
  не выкидывать целую нишу из-за одного 👎 — смотреть на `weight` и причину. Применить гео-фильтр на этапе выбора.
- **ШАГ 2 (скауты):** в промпт скаута добавить гео-правило и известные репеллеры («не предлагай X, потому что…»).
- **ШАГ 4 (red-team):** отдельный гейт — кандидат, нарушающий `geo`/`hard_filters` или явный репеллер,
  **отбрасывается или занижается**; совпадающий с аттрактором — **поднимается** (приоритет в дайджесте).

## НИКОГДА

- **НЕ выдумывать вкус.** `attractors`/`repellers`/`reasons` — только из реальных отметок экспорта. Нет экспорта —
  не сочинять предпочтения; работать по гео-политике и `excluded_niches` по умолчанию.
- **НЕ превращать один 👎 в жёсткий бан категории.** Один сигнал → `weight` низкий; бан — только при явном
  повторе или явной причине пользователя.
- **НЕ ломать схему `findings.json` и не трогать `index.html`-контракт** ради вкуса — петля живёт в
  `preferences.json` + промпте, гео остаётся правилом генерации (без нового поля в данных).

## Зачем

Без замкнутой петли прогон каждый раз ищет «вслепую» и повторяет то, что ты уже отметил «мимо». Экспорт + анализ
причин превращают твои клики в конкретные смещения поиска и в гео-фильтр — радар становится персональным.
